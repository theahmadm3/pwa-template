2. NEXT.CONFIG.TS - PWA Configuration
=================================

File: next.config.ts
Lines: 1-14 (entire file)

import nextPWA from "next-pwa";

const withPWA = nextPWA({
	dest: "public",
	register: true,
	skipWaiting: true,
	maximumFileSizeToCacheInBytes: 10 * 1024 * 1024,
});

const nextConfig = {
	// Your existing Next.js configuration here, if any
};

export default withPWA(nextConfig);

Description: Configures next-pwa with the following settings:
- dest: "public" - Service worker files will be generated in the public folder
- register: true - Automatically registers the service worker
- skipWaiting: true - Service worker will activate immediately
- maximumFileSizeToCacheInBytes: 10MB - Sets cache size limit

Lines: 19-21 (PWA state management)

	const [isAppInstalled, setAppInstalled] = useState(false);
	const [isMobile, setMobile] = useState(false);

Lines: 23-68 (PWA event handlers and lifecycle management)
	useEffect(() => {
		const handleBeforeInstallPrompt = (e: BeforeInstallPromptEvent) => {
			e.preventDefault();
			setDeferredPrompt(e);
		};

		const handleAppInstalled = () => {
			setAppInstalled(true);
		};

		const checkStandalone = () => {
			if (
				window.matchMedia &&
				window.matchMedia("(display-mode: standalone)").matches
			) {
				setAppInstalled(true);
			}
		};

		const checkMobile = () => {
			setMobile(window.matchMedia("(max-width: 768px)").matches);
		};

		window.addEventListener(
			"beforeinstallprompt",
			handleBeforeInstallPrompt as any,
		);
		window.addEventListener("appinstalled", handleAppInstalled);
		window.addEventListener("DOMContentLoaded", checkStandalone);
		window.addEventListener("load", checkStandalone);
		window.addEventListener("pageshow", checkStandalone);
		window.addEventListener("resize", checkMobile);
		checkMobile();

		return () => {
			window.removeEventListener(
				"beforeinstallprompt",
				handleBeforeInstallPrompt as any,
			);
			window.removeEventListener("appinstalled", handleAppInstalled);
			window.removeEventListener("DOMContentLoaded", checkStandalone);
			window.removeEventListener("load", checkStandalone);
			window.removeEventListener("pageshow", checkStandalone);
			window.removeEventListener("resize", checkMobile);
		};
	}, [deferredPrompt]);

Lines: 70-88 (PWA install function)
	const installPWA = () => {
		if (!deferredPrompt) {
			console.log("Installation prompt not available");
			alert("Installation prompt not available");
			return;
		}
		(deferredPrompt as BeforeInstallPromptEvent).prompt();
		(deferredPrompt as BeforeInstallPromptEvent).userChoice.then(
			(choiceResult) => {
				if (choiceResult.outcome === "accepted") {
					console.log("User accepted the install prompt");
					setAppInstalled(true);
				} else {
					console.log("User dismissed the install prompt");
				}
				setDeferredPrompt(null);
			},
		);
	};

=================================
SUMMARY
=================================

The PWA implementation consists of:

1. **Dependencies**: next-pwa package and type definitions in package.json
2. **Build Configuration**: next.config.ts configures PWA generation with next-pwa
3. **Type Definitions**: src/types/next-pwa.d.ts for TypeScript support
4. **Runtime Logic**: src/app/layout.tsx handles install prompts and PWA state
5. **Manifest**: public/manifest.json defines PWA metadata and appearance
6. **Assets**: Icon files and auto-generated service worker files

This setup enables the app to:
- Be installable on mobile and desktop devices
- Work offline (via service worker caching)
- Display as a standalone app when installed
- Show custom install prompts on supported browsers
- Provide native app-like experience with icons and splash screens
